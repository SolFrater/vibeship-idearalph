### 4. MCP Server (NEW!)
Built a full MCP server for auto-invocation during brainstorming:

```
mcp-server/
├── src/
│   ├── index.ts         # Server entry point
│   ├── tools.ts         # 5 MCP tools with handlers
│   └── ralph.ts         # Ralph engine (scoring, loop, PRD)
├── package.json
└── README.md
```

**MCP Tools:**

| Tool | Description |
|------|-------------|
| `idearalph_brainstorm` | Generate and score startup ideas for a topic |
| `idearalph_validate` | Score an idea on 10 PMF dimensions |
| `idearalph_refine` | Run Ralph Loop with modes: single, target, max |
| `idearalph_prd` | Generate PRDs at napkin/science-fair/genius levels |
| `idearalph_architecture` | Implementation plan + Spawner skills recommendation |

**Refinement Modes:**
- `single` - One iteration only
- `target` - Keep refining until score >= target (default 9.5)
- `max` - Run all iterations for maximum polish

**Configured in:** `~/.claude/mcp.json`

## Strategic Decisions

### Plugin-First Strategy
- Focus on CLI plugin experience, not website sync
- Ideas stay private in user's local environment
- Website becomes funnel TO the plugin, not replacement

### PRD Naming
- Rejected: basic/detailed/enterprise (too corporate)
- Rejected: valedictorian (not globally understood)
- Accepted: napkin/science-fair/genius (fun, universal, fits Ralph energy)

### MCP vs Plugin
- **Plugin:** Slash commands for explicit invocation (`/ralph-loop`, `/ralph-prd`)
- **MCP:** Auto-invokes when user brainstorms naturally
- Both installed and working!

### Spawner Integration
- IdeaRalph guides users to install Spawner after PRD generation
- 450+ specialized skills (not just code - marketing, strategy, etc.)
- Free version now, subscription model with premium skills later
